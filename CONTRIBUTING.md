# Contributing to Diskmap

Thank you for considering contributing to Diskmap! It's people like you that make open source such a great place.

Since this project is in its early stages and led by a learner, our rules are simple and flexible. We value your input and help!

## 🚀 Getting Started

To get started with development:

1. **Fork** the repository on GitHub.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/your-username/diskmap.git
   ```

## 🐛 Using the Issue Tracker

If you find a bug, please open an issue and include:

* **Version**: The version of Diskmap or the commit hash.
* **OS**: Your operating system and version.
* **Reproduction**: Clear steps to reproduce the behavior.
* **Logs**: Any relevant error messages or terminal output.

For feature requests, please provide a detailed description of what you'd like to see and why.

## 🛠️ Pull Requests

Good pull requests are a fantastic help! We follow the [GitHub Flow](https://docs.github.com/en/get-started/quickstart/github-flow).

1. Create a branch for your changes: `git checkout -b feature/my-new-feature` or `bugfix/fix-issue-x`.
2. Commit your changes (we recommend [Conventional Commits](https://www.conventionalcommits.org/)).
3. Push to your fork and submit a Pull Request to the `dev` branch.

**Before submitting, please ensure:**
* You have followed the [Coding Style](CODING_STYLE.md).
* Your code passes all tests (`cargo test`).
* You have run `cargo fmt` and fixed any `cargo clippy` warnings.

## 🤝 Code of Conduct

We are committed to providing a welcoming and inspiring community. Please be respectful and professional in all interactions. (Full Code of Conduct coming soon).

Don't hesitate to ask questions if you're not sure about anything. Happy coding!
